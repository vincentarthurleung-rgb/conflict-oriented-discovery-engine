# Validation Aggregator Benchmark

This deterministic rule benchmark prevents conservative status semantics from
drifting. Fixture signals cover supported, contradicted, mixed, no coverage,
missing index, insufficient quality, error, and the specific limits that trial
existence and binding records are not direct scientific proof. Cache miss is
kept separate from no coverage.

```bash
python -m code_engine.cli.benchmark_validation_aggregator \
  --cases tests/fixtures/validation_benchmark/benchmark_cases.jsonl \
  --output /tmp/validation_benchmark_report.json
```

The output reports case count, matched statuses, accuracy, and mismatches. It
uses no real data, network, or external API and must not be described as pilot
accuracy.
