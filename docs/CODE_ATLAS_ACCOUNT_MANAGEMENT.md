# C.O.D.E. Atlas Account Management

Owner access lives under `/owner/access`, `/owner/users`, and `/owner/security`.

The Owner can create `admin`, `developer`, `reviewer`, and `pharma` accounts. The UI does not offer `owner`; the database keeps the single enabled Owner invariant. Temporary passwords are generated by the server, returned only in the creation response, and stored only as password hashes.

Users are not hard-deleted. Disabling an account preserves annotations, audit events, adjudications, Gold provenance, metric provenance, and historical username/display-name snapshots.

Owner recovery is intentionally a CLI operation when web/email recovery is unavailable. Use the existing user-admin CLI against `data/code_atlas.db`; do not add a second owner through the web UI.
